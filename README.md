# App Accessibility Checker (Figma Plugin)

앱 접근성 가이드에 맞춰 다음 세 가지 항목을 자동 점검합니다.

1. **색에 무관한 콘텐츠 인식** — 인디케이터처럼 동일한 형태가 색상만 달라지는 패턴을 찾아 비색 구분 수단을 권장합니다.
2. **텍스트/이미지 콘텐츠 명도 대비** — 텍스트 및 단일 도형(아이콘 등)의 전경·배경 대비가 3:1 이상인지 확인합니다.
3. **자동으로 변경되는 콘텐츠 제어** — 명칭/텍스트에 따라 자동 슬라이드 배너로 추정되는 프레임에 이전/다음/정지/전체보기 컨트롤이 있는지 검사합니다.

## 설치 방법

1) Figma Desktop에서: Plugins → Development → Import plugin from manifest…
2) 현재 폴더의 `manifest.json` 선택
3) 캔버스에서 Plugins → Development → App Accessibility Checker 실행

## 사용법

1) Scope(선택/현재 페이지/문서 전체)와 점검 규칙을 선택합니다.
2) 필요 시 명도 대비 기준(기본 3:1)을 조정합니다.
3) Run checks를 실행하고 결과 목록의 Select 버튼으로 해당 노드를 선택/줌 합니다.

## 한계 및 참고

- 대비 체크는 부모 프레임의 단색(SOLID) 또는 화이트 배경을 가정합니다. 복잡한 배경/이미지에서는 정확도가 떨어질 수 있습니다.
- 색 비의존 검출은 3개 이상의 동일 크기 도형(인디케이터 등)을 대상으로 하며, 실제 상태 변화 여부는 디자이너가 추가로 확인해야 합니다.
- 자동 콘텐츠 검출은 프레임 이름/텍스트에 대한 키워드 매칭을 활용합니다. 명명 규칙이 다르면 누락될 수 있으니 참고하세요.
